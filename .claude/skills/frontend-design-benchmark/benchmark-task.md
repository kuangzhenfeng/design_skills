## 一、设计目标

为虚构产品 **「Aurora 智能分析平台」** 设计主控 Dashboard。
Aurora 是一个面向数据团队的实时数据观测与告警平台，用户每天打开它查看核心指标、排查异常、处理告警。

你需要交付**这一个页面**的完整实现，要求：

- 视觉上有强烈的设计感与品质感，告别通用 AI 审美
- 信息架构清晰，数据可读性强
- 完整可交互、可运行
- 在 1440px 桌面端为主，同时需兼顾 1280px / 1920px，并支持平板与移动端的自适应

## 二、必须包含的模块（共 8 个）

| # | 模块 | 必须实现的功能点 |
|---|------|----------------|
| 1 | **左侧导航栏** | Logo + 至少 6 个导航项（含分组/分隔）+ 当前项高亮 + 可折叠（折叠态保留图标）+ 底部用户区 |
| 2 | **顶部栏** | 面包屑或页面标题 + 全局搜索（带 ⌘K 触发与建议下拉）+ 通知铃铛（未读红点/角标 + 下拉面板）+ 主题切换（亮/暗）+ 用户头像菜单 |
| 3 | **KPI 指标卡 ×4** | 每张卡：大数字 + 同比/环比涨跌（带方向箭头与红绿色）+ 迷你趋势图（sparkline）+ 悬停浮起；4 张卡分别为「活跃用户 / 数据吞吐量 / 异常事件 / 平均响应延迟」，其中「异常事件」需有警告态视觉 |
| 4 | **主图表区** | 一个可切换时间范围（24h / 7d / 30d / 90d）的折线/面积图，支持图例、hover tooltip、平滑曲线，有明确的视觉主次 |
| 5 | **次级图表** | 一个环形/饼图（事件类型分布）+ 一个横向条形图或排行榜（Top 5 数据源），二者并排或上下排布 |
| 6 | **数据表格 / 项目列表** | 一个带状态标签（在线/离线/异常）、进度条、负责人头像、最后更新时间的列表，支持行 hover 高亮，至少 6~8 行真实感数据 |
| 7 | **活动时间线** | 一条垂直时间线，展示最近活动，带时间戳、操作类型图标、操作人，有"加载更多"占位 |
| 8 | **告警 / 状态面板** | 一个独立卡片或抽屉，列出当前告警，含严重等级（critical/warning/info 三级配色）、可"忽略"/"处理"操作按钮、实时感（脉冲/呼吸光晕等状态动效） |

> 8 个模块都要真实落地，不允许用"TODO"/占位文字代替。数据使用 mock，但要真实可信、数值自洽。

## 三、设计要求（评分看点）

### 视觉与设计系统
- 统一的设计 token（颜色、字号、圆角、间距、阴影、模糊），不要散落硬编码
- 一个鲜明且自洽的主题色方案（允许你自由发挥品牌色，但需克制、有层次，主色 + 中性色 + 语义色体系完整）
- 暗色模式必须真正可用（对比度达标，不是简单反色）
- 字体层级清晰（标题/正文/辅助文字/数据数字有明确区分，数据数字建议用 tabular-nums 或等宽特性）

### 动效体系
- 页面需有一套**完整且有体系的动效**，覆盖至少以下场景，且保持流畅：
  - 页面/模块**入场动效**（错落浮现等）
  - KPI 卡与图表的**加载/数字动效**（数字滚动、骨架或流光）
  - 悬停/点击等**指针动效**（浮起、涟漪、光泽跟随等）
  - 状态类**常驻动效**（在线光晕、呼吸、告警脉冲）
  - Tab/时间范围切换的**过渡动效**
- 动效只动 `transform` / `opacity` / `filter`（合成层），不得动画化 `box-shadow` / `width` / `top` 等触发重排重绘的属性，保证 60fps 不掉帧

### 组件完成度与交互
- 所有可交互元素必须有完整状态（default / hover / active / focus / disabled）
- 表单控件（搜索、下拉、开关）真实可用，有 focus ring
- 主题切换**真实生效**，整页随之变化
- 无障碍：键盘可达、焦点可见、语义化标签、图标有 aria-label

### 响应式
- 桌面（≥1280px）/ 平板（768~1279px）/ 移动（<768px）三档都要合理布局
- 移动端导航需折叠为抽屉或底部栏，图表与卡片自适应堆叠

### 材质与质感
- 鼓励有设计语言层面的材质表达（玻璃拟态 / 渐变光晕 / 微噪点 / 镜面边缘高光 / 微妙阴影层次等，按你的设计语言自由取舍），但要服务于信息层级，不为炫技而堆砌

## 四、技术栈约束（必须严格遵守，不得替换）

| 项 | 要求 |
|----|------|
| 语言 | **TypeScript** |
| 框架 | **Next.js**（基于 **React**，App Router）—— 本题锁死用 Next.js，**不得**用 Vite/CRA 等其他脚手架 |
| 样式 | **Tailwind CSS** |
| 组件库 | **shadcn/ui**（基于 **Radix UI** 原语搭建）作为无障碍/交互基座——两者都要用，shadcn 的组件 + Radix 原语 |
| 动效 | **Motion**（即 Framer Motion，`motion/react` 或 `framer-motion`） |
| 图表 | 可用任意主流库（recharts / visx / 自己用 SVG 手绘均可），但不得引入与你设计语言冲突的默认丑样式——要么深度定制，要么手绘 |
| 数据 | 全部 mock，写在前端，禁止调用真实 API |

> **技术栈铁律**：TypeScript + React + Next.js + Tailwind CSS + shadcn/ui + Radix + Motion **七项缺一不可、不得替换**。这是公平对比的统一基线，任何一项缺失或被替换（如用 Vite 代替 Next.js、只用 Radix 而不用 shadcn）都视为违反技术栈约束。

> **交付物**：一个可一键启动、在浏览器打开即看到完整效果的工程。
> 项目根目录需有清晰的 `README`（含启动命令）和标准的 **Next.js** 工程（`npm install && npm run dev` 零配置启动，默认 `http://localhost:3000`）。
